Редактор кода Cursor, модель composer 2

# 2) Описание сделанного и проверка инструкций

### Краткое описание сделанного
1. README — описание назначения, сценариев, стека, запуска и структуры `src/` (FSD).
    
2. `.cursorrules` — правила для агента: FSD, стиль, план до кода, проверки, без лишних зависимостей и ломки архитектуры, формат ответа (план → файлы → зачем → validation).
    
3. `docs/ai/` — набор для AI: `CONTEXT`, `ARCHITECTURE`, `STYLEGUIDE`, `TASKS` (примеры промптов), `DEFINITION_OF_DONE`.
    
4. «Новый чат» — в сайдбере кнопка, список пользователей из стора; `openOrCreateDirectChat` в Zustand (новый direct или переход в существующий), фича `features/new-chat`.
    
5. Баг после «Очистить историю» — при очистке активного чата сбрасывается поиск по сообщениям (закрытие панели, пустой запрос, индекс 0), без правок UI поиска.
    
6. Рефакторинг ленты — разметка одного сообщения вынесена в `widgets/chat-panel/ui/MessageItem.tsx`, в `MessageList` остались скролл и список.
   
### Проверка задач   
##### 1) Feature
Задача: Добавь в боковую панель чатов кнопку «Новый чат»

Промпт:
Следуй docs/ai/CONTEXT.md и ARCHITECTURE.md.

Добавь в боковую панель чатов кнопку «Новый чат». По клику — простой UI без новых npm-пакетов: список пользователей из сида, кроме текущего (CURRENT_USER_ID). По выбору пользователя создаётся новый личный (direct) чат с пустой историей, он становится активным и отображается в списке с корректным заголовком. Если чат с этим собеседником уже есть — не дублируй, переключись на существующий.


Логика — в actions Zustand (messenger-store), UI — в feature или widget. Не меняй слои FSD, не подключай роутер.

Результат:
На выходе есть план, список измененных файлов, объяснение изменений и  шаги проверки

##### 2) Bugfix
Задача: Исправить баг с очисткой истории

Промпт:
Следуй docs/ai/DEFINITION_OF_DONE.md.

Баг: после «Очистить историю» в настройках чата состояние поиска по сообщениям (открытость панели, индекс совпадения, при необходимости запрос) ведёт себя некорректно относительно пустой ленты — исправь минимальным диффом в messenger-store (и при необходимости в UI поиска). Не рефактори стор целиком. В ответе: шаги воспроизведения, «было / стало».

Результат:
На выходе есть план, список измененных файлов, объяснение изменений и  шаги проверки
