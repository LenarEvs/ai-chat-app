# Definition of Done (для AI и ревью)

Документ задаёт, когда задача считается **завершённой**. Общий контекст: [CONTEXT.md](./CONTEXT.md).

---

## Обязательный формат ответа (при изменении кода)

1. **План** — краткие шаги.
2. **Изменяемые файлы** — пути + создание/правка/удаление.
3. **Объяснение** — зачем так, границы задачи (что сознательно не трогали).
4. **Шаги проверки** — см. разделы ниже.

---

## Автоматические проверки

| Когда | Команда | Критерий |
|--------|---------|----------|
| Любой код TS/TSX | `npm run lint` | Нет новых ошибок ESLint в изменённых файлах |
| Менялись импорты, типы, конфиг | `npm run build` | Сборка завершается успешно |

Если агент не может запустить команды в среде — явно перечислить, что выполнить **локально**.

---

## Ручная проверка UI

Запуск: `npm run dev`.

**Базовый смок** (если трогались чаты/сообщения/навигация):

1. Выбрать чат, отправить сообщение, дождаться мок-ответа.
2. Переключить чат, убедиться, что список и превью текста актуальны.
3. Открыть поиск по чатам / по сообщениям (если менялось).
4. Узкая ширина окна: переход «список ↔ чат», кнопка назад.

**Настройки** (если трогались):

- Экран «Настройки»: переключатели компактных пузырей и Enter.
- «Настройки чата»: заглушение, переименование группы (для групповых), очистка истории.

**Профиль** (если трогался):

- Смена отображаемого имени и username, отображение в шапке/аватарах без ошибок.

## Тестирование

- **По умолчанию** в проекте нет обязательного набора unit/e2e тестов в CI.
- DoD для обычной задачи: **lint + при необходимости build + ручной смок** (см. выше).
- Если в постановке явно сказано «добавить тесты»:
  - выбрать минимальный инструмент **без лишних зависимостей** или по согласованию с владельцем;
  - тесты должны быть **стабильными** и отражать баг/фичу, а не копировать снапшот всего DOM.

---

## Ограничения по объёму (anti-scope-creep)

Задача **не считается закрытой**, если вместе с ней непросимно:

- добавлены новые крупные слои каталогов;
- заменён стейт-менеджмент или фреймворк;
- подключены тяжёлые библиотеки без запроса;
- выполнен несвязанный массовый реформат файлов.

Такие изменения нужно **откатить** или вынести в отдельную задачу.

## Документация

- Если меняется **поведение продукта** (новые сценарии, ограничения): обновить [README.md](../../README.md) или `docs/ai/*` **только если** это запрошено в задаче.
- Исправление опечаток в docs по пути — с пользой, но не вместо исправления кода.

---

## Готовность к merge (чеклист)

- [ ] План и список файлов были до правок (или согласовано иначе).
- [ ] `npm run lint` ок (или перечислены ограничения среды).
- [ ] `npm run build` ок, если нужен по критериям выше.
- [ ] Ручной смок пройден / описан для пользователя.
- [ ] Нет лишних файлов, отладочных `console.log`, закомментированных кусков без причины.
