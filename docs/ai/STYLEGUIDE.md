# Code style и соглашения (для AI)

## TypeScript

- Строгая типизация; **избегать `any`**. Для «ещё неизвестно» — `unknown`, узкие type guards, дженерики.
- Публичные пропсы компонентов и контракты store — **явно типизированы**.
- Не ослаблять `eslint`/`tsc` без обоснования в задаче.

## Импорты и модули

- Использовать алиас **`@/`** для путей из `src/` (как в существующем коде).
- Предпочитать **именованные** экспорты; публичные входы сегментов — через `index.ts`, если так уже сделано в соседних фичах.

## React

- Только **функциональные** компоненты.
- Соблюдать правила хуков (`react-hooks` из ESLint).
- Не оставлять неиспользуемых импортов и мёртвого кода после правок.

## Стили (Tailwind)

- Утилитарные классы в `className`; длинные строки — как в соседних компонентах (переносы для читаемости).
- Условные классы — через **`cn`** из `@/shared/lib/cn`, не дублировать свою обёртку без нужды.
- Цвета/отступы — в духе текущей тёмной темы мессенджера; не вводить произвольную вторую дизайн-систему.

## Именование

- Компоненты и типы компонентов: **PascalCase**.
- Функции, переменные, поля стора: **camelCase**.
- Файлы UI: согласовать с проектом (`PascalCase.tsx` для компонентов, `kebab-case` для папок сегментов — как в репозитории).

## Комментарии

- Кратко, по делу.
- **Язык комментариев в файле** — не смешивать стили без причины: в проекте уже встречаются русские JSDoc в некоторых файлах; в новом коде придерживаться доминирующего стиля **файла**, который правишь.

## Линтинг и форматирование

- После изменений: **`npm run lint`**; исправлять новые предупреждения/ошибки в затронутых файлах.
- Не отключать правила точечно (`eslint-disable`) без краткого комментария «почему».

## Зависимости

- **Не добавлять** npm-пакеты без необходимости.
- Если добавление неизбежно — в ответе: зачем пакет, альтернатива без него, влияние на размер бандла.

## Доступность и разметка

- Сохранять осмысленные **`aria-`**, `role`, подписи кнопок там, где они уже есть.
- Не ухудшать фокус и клавиатурную навигацию при правках интерактива.
