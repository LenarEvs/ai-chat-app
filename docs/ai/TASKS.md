# Примеры задач для AI

Используй как шаблоны постановки. Контекст проекта: [CONTEXT.md](./CONTEXT.md).

## Сценарий A: добавление новой функции

### Пример формулировки

> Добавь в боковой список чатов кнопку «Создать чат с…»: по клику открывается простой диалог с выбором пользователя из сида (кроме текущего). После подтверждения создаётся новый **личный** чат с пустой историей и сразу активируется. Без новых npm-зависимостей, соблюдай FSD.

### Ожидания от исполнителя

- План и список файлов до кода.
- Логика создания чата и обновление списка — в **`useMessengerStore`** (или новый action рядом), типы в `entities/chat`.
- UI диалога — в подходящем **widget** или **feature** (не раздувай `pages` логикой).
- Проверки: `lint`, `build`, ручной сценарий — создать чат, отправить сообщение, проверить сортировку по времени.

## Сценарий B: исправление бага

### Пример формулировки

> Баг: при узкой ширине экрана после возврата из открытого чата к списку счётчик непрочитанных для чата X иногда не совпадает с ожидаемым (опиши шаги: открыть чат A, получить мок-ответ в фоне для чата B, нажать «назад»…). Найди причину в `messenger-store` или в навигации `MessengerPage` / `ChatPanel`, исправь минимальным диффом. Напиши «было / стало».

### Ожидания от исполнителя

- Воспроизведение шагами из задачи или уточнение, если шаги неполные.
- **Не** рефакторить стор целиком; точечный фикс + краткое объяснение гонки/условия.
- Validation: `lint`, при затронутых типах — `build`; ручная проверка на desktop и narrow.

## Сценарий C: рефакторинг

### Пример формулировки

> Вынеси разметку строки сообщения из `MessageList` в маленький подкомпонент `MessageBubble` в том же виджете (или сегменте `ui/`), прокинь только нужные пропсы. Поведение и классы Tailwind должны совпасть пиксельно с текущим UI. Не меняй публичный API стора.

### Ожидания от исполнителя

- План: что выносится, какие пропсы, что **не** меняется (store, типы сообщений).
- Дифф в основном в **одном виджете** или связанных `ui`-файлах; без смены слоёв FSD.
- Validation: `lint`, `build`; быстрая ручная проверка: длинные сообщения, поиск по чату, компактные пузыри.

## Дополнительные короткие примеры

| Тип | Формулировка |
|-----|----------------|
| Копирайт | Обновить тексты мок-ответов бота в `mock-backend.ts`, не ломая ветвление `isBot`. |
| UX | Добавить `aria-label` для кнопки закрытия поиска в шапке чата. |
| Данные | Расширить сид новым групповым чатом и 2 сообщениями; обновить типы только если нужно. |
