## Feature (новая функция)

### Пример промпта

```text
Следуй docs/ai/CONTEXT.md и ARCHITECTURE.md.

Добавь в боковую панель чатов кнопку «Новый чат». По клику — простой UI без новых npm-пакетов: список пользователей из сида, кроме текущего (CURRENT_USER_ID). По выбору пользователя создаётся новый личный (direct) чат с пустой историей, он становится активным и отображается в списке с корректным заголовком. Если чат с этим собеседником уже есть — не дублируй, переключись на существующий.

Логика — в actions Zustand (messenger-store), UI — в feature или widget. Не меняй слои FSD, не подключай роутер.
```

### Ожидаемый результат

- В интерфейсе доступно действие «Новый чат»; выбор пользователя создаёт/активирует диалог, пустая история, отправка и мок-ответ работают.
- Логика в `useMessengerStore` (или типах `entities/chat`), без второго глобального стора; дубликатов direct-чатов с тем же peer нет.
- `npm run lint` и `npm run build` проходят; вручную проверены создание чата и повторный выбор того же пользователя (переход в существующий чат).

## Bugfix (исправление бага)

### Пример промпта

```text
Следуй docs/ai/DEFINITION_OF_DONE.md.

Баг: после «Очистить историю» в настройках чата состояние поиска по сообщениям (открытость панели, индекс совпадения, при необходимости запрос) ведёт себя некорректно относительно пустой ленты — исправь минимальным диффом в messenger-store (и при необходимости в UI поиска). Не рефактори стор целиком. В ответе: шаги воспроизведения, «было / стало».
```

### Ожидаемый результат

- После очистки истории поиск не ссылается на несуществующие сообщения: счётчики/подсветка/активное совпадение согласованы с пустым списком сообщений (или поиск закрыт — если так зафиксировано в ответе агента).
- Изменения точечные, без новых зависимостей; `npm run lint`, при затронутых типах — `npm run build`.
- В описании задачи от агента есть воспроизведение и «было / стало».

## Refactoring (рефакторинг)

### Пример промпта

```text
Вынеси разметку одного сообщения (пузырь, своё/чужое, подсветка поиска) из MessageList в компонент MessageItem.tsx в каталоге widgets/chat-panel/ui/. Прокинь только нужные пропсы. Поведение и классы Tailwind — как сейчас (включая compactBubbles). Скролл и группировку по датам не переноси. API стора и типы Message не меняй.
```

### Ожидаемый результат

- Есть `MessageItem.tsx` (или эквивалент в том же `ui/`), `MessageList` только композирует список; визуально поведение не изменилось.
- `messenger-store` и `entities/message` не менялись без необходимости; слои FSD сохранены.
- `npm run lint`, `npm run build`; ручная проверка: длинное сообщение, поиск с совпадениями, режим компактных пузырей.
